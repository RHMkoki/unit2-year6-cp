Exam-Style Questions

Q1: What is wrong with this algorithm?

Enjoy drink → Pour drink into glass → Put away drink and glass → Open bottle of drink

A) It is impossible
B) It is missing a loop
C) It is out of sequence
D) Nothing at all

Answer: C - You can't enjoy a drink before opening the bottle!

Q2: What does "being efficient" mean when writing algorithms?

A) Using the fewest number of commands to achieve the same outcome
B) Making sure you include selection
C) Using as many commands as possible
D) Making sure you remove indentations from the algorithm

Answer: A - Efficiency means achieving the same result with fewer commands.

Lesson 4: From Algorithms to Code

Core Concepts

Algorithm vs Code: An algorithm uses agreed notation; code is exactly what the computer reads.

Scratch Terminology: In Scratch, we use "repeat" instead of "loop".

Sequencing: Commands must be in the correct order for the program to work properly.

Code Command Repeat Scratch Sprite Debugging Notation

Key Activity: Scratch Translation

Translate your robot algorithm into Scratch code:

  1. Use motion blocks for movement
  2. Use control blocks for repetition
  3. Test and debug your code

Scratch Code Example

when green flag clicked

repeat (5)

  move (10) steps

  turn cw (15) degrees

  wait (1) seconds

go to x:(0) y:(0)

This program moves, turns, waits, then returns to center

move (10) steps

Core Concepts

Game Design: Breaking down a complex game into manageable parts.

Sprites: Characters or objects in a game (like the frog and cars).

Decomposition: Breaking a big problem into smaller sub-problems.

String: A sequence of characters (text) used to display messages.

Lesson 6: Forever Loops & Movement

Core Concepts

Forever Loop: A loop that runs continuously until the program stops.

Automated Movement: Using loops to make sprites move without user input.

Multiple Sprites: Programming each sprite separately with its own code.

Stage Coordinates: The center is (0,0); x-axis is horizontal, y-axis is vertical.

Forever Loop Obstacle Sprite Coordinates Automated Movement x-axis / y-axis

Key Activity: Programming Car Movement

Add car sprites and program them to move automatically:

  1. Add car sprite using "Choose a Sprite"
  2. Program with forever loop for continuous movement
  3. Set starting position with coordinates
  4. Adjust speed by changing step amount

Sample Car Movement Code

when green flag clicked

forever

  go to x: (230) y: (-130)

  repeat (60)

    change x by (-8)

  hide

  wait (3) seconds

  show

This makes a car move left across the screen, hide, wait, then reappear

Lesson 7: IF...THEN Conditions for Interaction

Core Concepts

Conditional Statements: IF...THEN blocks that check conditions and run code when true.

Interaction: When sprites touch or collide with each other.

Input & Output: Input = user action or collision, Output = game response.

Sensing Blocks: Detect when sprites touch or conditions are met.

IF...THEN... Condition Interaction Input Output Sensing Block Decompose

Key Activity: Programming Collision Detection

Program what happens when the frog touches a car:

  • Game stops
  • Sprite changes costume (to "frog splat")
  • Play "splat" sound effect
  • Wait 3 seconds, then reset to starting position

Collision Detection Code

when green flag clicked

forever

  if touching (Car Sprite) ? then

    switch costume to (Frog Splat)

    play sound (Splat Noise) until done

    wait (3) seconds

    go to x: (-167) y: (7)

    switch costume to (Frog Normal)

The forever loop constantly checks if frog touches car, then runs the collision code

IF...THEN Logic Flow

Condition Check: Frog touches car?
↓
IF TRUE:
Change costume, play sound, reset
↓
IF FALSE:
Continue game normally

Conditional statements create different pathways in your program
